# 기능 정의
- 메인 홈 (대시보드)
- 인증 기능
	- 로그인
	- 회원가입
	- 비밀번호 찾기
- 식사 기록
	- 식사 추가 기능

# React 라우터 표현
```
/                 메인 홈 (대시보드)

/login            로그인
/signup           회원가입
/forgot           비밀번호 찾기

/meal/add         식사 기록 추가
```

---

## 1. 메인 홈
경로: `/`
목적: 앱 진입 시 보이는 메인 화면 (대시보드)

### 화면 요소
- 상단: 서비스 로고 및 네비게이션
- 오늘의 칼로리/영양소 요약 정보
- 최근 식사 기록 목록
- 식사 추가 버튼

### 사용자 행동
- 식사 추가 버튼 클릭 → `/meal/add`
- 로그인 안 된 상태일 경우 → `/login` 으로 리다이렉트 (필요 시)

---

## 2. 로그인
경로: `/login` (실제 폴더: `src/app/(auth)/login`)
목적: 사용자가 계정에 로그인하기 위함

### 화면 요소
- 입력
	- 이메일 (또는 아이디)
	- 비밀번호
- 버튼
	- 로그인
	- 회원가입 페이지로 이동
	- 비밀번호 찾기로 이동

### 사용자 행동
- 로그인 클릭 → 인증 처리 후 메인 홈(`/`)으로 이동
- 회원가입 클릭 → `/signup` 이동
- 비밀번호 찾기 클릭 → `/forgot` 이동

### 필요 데이터
- email
- password

---

## 3. 회원가입
경로: `/signup` (실제 폴더: `src/app/(auth)/signup`)
목적: 신규 사용자의 서비스 가입

### 화면 요소
- 입력
	- 이메일
	- 비밀번호
	- 비밀번호 확인
- 버튼
	- 회원가입
	- 이미 계정이 있으신가요? (로그인으로 이동)

### 사용자 행동
- 회원가입 버튼 클릭 → 계정 생성 후 로그인 화면(`/login`)으로 이동
- 로그인으로 이동 클릭 → `/login` 이동

### 필요 데이터
- email
- password
- passwordConfirm

---

## 4. 비밀번호 찾기
경로: `/forgot` (실제 폴더: `src/app/(auth)/forgot`)
목적: 비밀번호 분실 시 재설정을 위한 기능

### 화면 요소
- 입력
	- 이메일
- 버튼
	- 재설정 메일/링크 발송
	- 로그인으로 돌아가기

### 사용자 행동
- 발송 버튼 클릭 → 이메일 발송 안내 메시지 노출
- 뒤로가기 클릭 → `/login` 이동

### 필요 데이터
- email

---

## 5. 식사 기록 추가
경로: `/meal/add`
목적: 새로운 식사 내용과 칼로리를 기록

### 화면 요소
- 상단: 페이지 제목 "식사 추가"
- 입력
	- 음식 검색 또는 이름 직접 입력
	- 섭취량 및 칼로리 정보
	- 섭취 시간/날짜 (선택)
- 버튼
	- 등록
	- 취소

### 사용자 행동
- 등록 클릭 → 식사 기록 저장 후 홈(`/`)으로 이동
- 취소 클릭 → 이전 화면으로 돌아가기

### 필요 데이터
- mealName (음식 이름)
- calories (칼로리)
- carbs (탄수화물)
- protein (단백질)
- fat (지방)
- mealTime (섭취 시간, 필요 시)
